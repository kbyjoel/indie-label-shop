# Skill: Stimulus + Select2

## Core problem

Select2 replaces the native `<select>` with a custom UI and triggers changes via jQuery:

```js
jQuery(select).trigger('change')
```

**`document.addEventListener('change', ...)`** and **`select.addEventListener('change', ...)`** do NOT catch these jQuery events. You must use jQuery `.on('change', ...)`.

---

## Pattern: listening to a Select2 from a Stimulus controller

```js
import { Controller } from '@hotwired/stimulus';

export default class extends Controller {
    connect() {
        this.doSomething();

        const select = document.querySelector('[data-my-select]');
        if (!select) return;

        if (window.jQuery) {
            this._handler = () => this.doSomething();
            window.jQuery(select).on('change', this._handler);
        } else {
            this._handler = () => this.doSomething();
            select.addEventListener('change', this._handler);
        }
    }

    disconnect() {
        const select = document.querySelector('[data-my-select]');
        if (!select || !this._handler) return;

        if (window.jQuery) {
            window.jQuery(select).off('change', this._handler);
        } else {
            select.removeEventListener('change', this._handler);
        }
    }

    doSomething() {
        const select = document.querySelector('[data-my-select]');
        const value = select ? select.value : null;
        // ...
    }
}
```

### Identifying the select without relying on Symfony's generated name

Add a neutral `data-` attribute on the `<select>` via the FormType:

```php
->add('myField', ChoiceType::class, [
    'attr' => ['data-my-select' => 'true'],
])
```

Then use `document.querySelector('[data-my-select]')` in the controller. This avoids depending on the name generated by Symfony (`my_form_type[myField]`).

## Pattern: cross-scope controller (select and targets in different tabs)

When `data-controller` cannot wrap both the select and the targets (e.g. they are in separate tabs and a wrapper div would break the tab system), place the controller on one side and find the other via `document.querySelector`.

```twig
{# Tab 1 — contains the select, NOT the data-controller #}
<div class="tab-pane active" id="panel-tab-general">
    {{ form_row(form.myField) }}   {# select with data-my-select="true" #}
</div>

{# Tab 2 — contains the controller and the targets #}
<div class="tab-pane" id="panel-tab-config"
     data-controller="my-controller">
    <div data-my-controller-target="block">...</div>
</div>
```

```js
// The controller is on tab 2, but finds the select in tab 1
connect() {
    const select = document.querySelector('[data-my-select]');
    // attach via jQuery...
}
```

> Never wrap `.tab-pane` elements in a `<div data-controller>` — it breaks the AropixelAdminBundle tab navigation system.

---

## Mandatory cleanup in `disconnect()`

Always remove jQuery listeners in `disconnect()` to avoid memory leaks, especially with Turbo (SPA navigation):

```js
disconnect() {
    const select = document.querySelector('[data-my-select]');
    if (select && window.jQuery && this._handler) {
        window.jQuery(select).off('change', this._handler);
    }
}
```
